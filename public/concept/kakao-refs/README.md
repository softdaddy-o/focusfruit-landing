## AB Test Axes

1. Clean cute vs grubby cute
   - Clean cute: simple outlines, soft body, readable expressions.
   - Grubby cute: tired face, messy texture, slightly strange proportions.

2. Animal likeness vs mascot abstraction
   - Animal likeness keeps the meerkat nose, eye mask, posture, and tail clear.
   - Mascot abstraction can push the head-body ratio and expression language harder.

3. Calm default vs high-reaction sticker energy
   - Calm default is important for a desktop companion that stays visible.
   - High-reaction cuts can be reserved for focus start, break, typing, scroll, and session end.

4. Large character vs mini readability
   - The floating desktop character must read at small sizes.
   - Mini-emoticon references are useful for deciding how much detail to remove.

5. Cute vs weird-cute
   - Cute should be the default product-safe direction.
   - Weird-cute can be tested as an optional expression pack or notification motion.

## Recommendation For Next Concept Round

Make 4 lanes for AD review:

1. `A Clean Long-Neck Meerkat`: safest main character, long neck, side-glance face.
2. `B Grubby Long-Neck Meerkat`: same silhouette, messier texture and tired expression.
3. `C Deadpan Meerkat`: blank stare, long body, low motion, strongest "하찮음".
4. `D High-Reaction Meerkat`: exaggerated mouth/eyes for typing, scroll, break, and alert states.
